## 1. Executive Summary

A PowerShell Script Block Logging event (Windows Event ID 4104) was detected on `win-endpoint` indicating the execution of a Base64-encoded PowerShell payload via `Invoke-Command`. Wazuh rule 91822 fired at severity level 12. The script block contained an `Invoke-AtomicTest` call consistent with adversary simulation tooling. The Shuffle SOAR playbook automatically notified the analyst via email within seconds of detection.

> **Note:** This was a controlled Atomic Red Team simulation (T1059.001-1) for detection engineering purposes. No actual malicious activity occurred.

## 7. Containment Steps

> In a real incident, execute these immediately upon alert:

1. **Isolate** win-endpoint from the network (disable NIC or quarantine via EDR)
2. **Kill** any suspicious PowerShell processes (`Get-Process powershell | Stop-Process`)
3. **Review** Windows Event Log 4104 for additional script blocks in the same session
4. **Decode** any Base64 strings found in script blocks
5. **Check for persistence** — review Run keys, scheduled tasks, startup folders
6. **Reset credentials** for any accounts active on the endpoint at time of execution
7. **Escalate** if lateral movement indicators found (Event ID 4624/4625, 4648)

## 8. Remediation

| Action | Priority | Notes |
|---|---|---|
| Enable PowerShell Constrained Language Mode | High | Limits PowerShell capabilities for non-admin users |
| Block Base64-encoded PowerShell via AppLocker/WDAC | High | Prevent execution of obfuscated scripts |
| Alert on `Invoke-Command` + Base64 combination | Medium | High-fidelity detection for this pattern |
| Review PowerShell execution policy | Medium | Should be `RemoteSigned` or stricter |
| Enable AMSI (Antimalware Scan Interface) | High | Scans script content before execution |

---

## 9. Lessons Learned

- **Script Block Logging (Event ID 4104)** provides full visibility into PowerShell execution — including decoded Base64 content — before the script runs. This is one of the most valuable Windows log sources for detecting PowerShell abuse.
- **Wazuh rule 91822** fired within 2 seconds of execution — detection latency is excellent.
- **SOAR automation** reduced analyst notification time to near-zero. Without automation, this alert would sit in a SIEM queue until manually reviewed.
- **Base64 encoding** is a common evasion technique (T1140) — detecting it in combination with PowerShell is a high-fidelity signal.
